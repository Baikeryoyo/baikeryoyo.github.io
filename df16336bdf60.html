常量

常量可以理解为值不变的量。常量值被定义后，在脚本的其他任何地方都不能改变

一个常量由英文字母，下划线，数字组成，但是数字不能作为首字母出现

常量一般以大写字母为主，便于在语言中和变量区分

按照正则表达式，它将被表述为^[a-zA-Z_\x80-\xff][a-zA-Z0-9_\x80-\xff]*$

在PHP中使用define()函数来定义全局常量

在PHP中使用const来定义类常量

define()的常量不能在类中使用，const可以在类中使用

define()可以在条件语句中定义常量，const不可以在条件语句中定义常量

const是语言结构，define()是一个方法，const比define()在编译时要快

访问常量

在PHP中，通常使用echo直接输出常量，也可以通过函数constant()来访问常量

魔术常量

魔术常量属于特殊的预定义常量，它们的值随着它们在代码中的位置改变而改变。

魔术常量通常以两个下划线__开头，并以两个下划线__结尾

魔术常量 说明
__LINE__ 文件中的当前行号
__FILE__ 文件的完整路径和文件名
__DIR__ 文件所在的目录
__FUNCTION__ 当前函数或方法的名称。匿名函数则为 {closure}
__CLASS__ 当前类的名称。类名包括其被声明的作用域
__TRAIT__ Trait 的名字。Trait 名包括其被声明的作用域
__METHOD__ 类的方法名或者类名
__NAMESPACE__ 当前命名空间的名称
ClassName::class 完整的类名

变量

变量来源于数学，是计算机语言中能储存计算结果或者能表示抽象概念。

变量可以通过变量名访问，在指令式语言中，变量通常是可变的。

PHP中的变量用一个美元符号后面跟变量名来表示，变量名是区分大小写的

一个变量由英文字母，下划线，数字组成，但是数字不能作为首字母出现

按照正则表达式，它将被表述为^[a-zA-Z_\x80-\xff][a-zA-Z0-9_\x80-\xff]*$

作用域

PHP有四种不同的变量作用域，作用域内的变量不能在另一个作用域中访问

  • local（局部作用域）
  • global（全局作用域）
  • static（静态作用域）
  • parameter（函数参数作用域）

local局部作用域，用于函数内部声明的变量，只能在函数内部访问

global全局作用域，用于函数外部声明的变量，只能在函数外部访问，

函数内部访问需要关键词global，或者数组$GLOBALS[index]

static静态作用域，用于在函数执行完毕之后保留某个局部变量

parameter函数参数作用域，变量在函数声明中，参数是通过调用函数的值转递给局部变量

可变变量

如果一个变量保存的值刚好是另一个变量的名字，那么可以直接通过访问一个变量得到另外一个变量的值

变量传值

将一个变量赋值给另一个变量

变量传值一共有两种方式：值传递，引用赋值

值传递

将变量保存的值赋值给另外一个变量保存，两个变量之间没有关系

引用赋值

PHP 支持引用赋值，引用赋值意味着两个变量指向了同一个数据，没有拷贝任何东西

$_GET

HTTP GET变量

在 PHP 中，预定义的 $_GET 变量用于收集来自 method=’get’ 的表单中的值

从带有 GET 方法的表单发送的信息，对任何人都是可见的（会显示在浏览器的地址栏），并且对发送信息的量也有限制

$_POST

HTTP POST变量

在 PHP 中，预定义的 $_POST 变量用于收集来自 method=’post’ 的表单中的值

从带有 POST 方法的表单发送的信息，对任何人都是不可见的（不会显示在浏览器的地址栏），并且对发送信息的量也没有限制

默认情况下，POST方法的发送信息的量最大值为 8 MB，可通过设置 php.ini 文件中的 post_max_size 进行更改

$_FILES

HTTP 文件上传变量

通过 HTTP POST 方式上传到当前脚本的项目的数组

该数组有以下元素

元素 说明
$_FILES[filename][name] 储存了上传文件的上传名
$_FILES[filename][size] 存储了上传文件的大小，单位为字节
$_FILES[filename][tmp_name] 文件上传的临时文件
$_FILES[filename][type] 上传文件的类型
$_FILES[filename][error] 存储了上传结果，返回0说明上传成功

$_REQUEST

HTTP Request变量，用来收集通过 GET 和 POST 方法发送的表单数据

预定义的 $_REQUEST 变量包含了 $_GET、$_POST 和 $_COOKIE 的内容

$_SESSION

PHP session 变量用于存储关于用户会话（session）的信息，或者更改用户会话（session）的设置。Session 变量存储单一用户的信息，并且对于应用程序中的所有页面都是可用的

$_COOKIE

HTTP Cookies

Cookie实际上是一个很小的文本文件，网站通过向用户硬盘中写入一个Cookie文件来标识用户。当用户下次再访问该网站时，浏览器会将Cookie信息发送给网站服务器，服务器通过读取以前写入的Cookie文件中的信息，就能识别该用户

Cookie的两种形式：
（1）会话Cookie。临时性的，只在浏览器打开时存在，主要用来实现Session技术。
（2）永久Cookie。永久性的，保存在用户硬盘上并在有效期内一直可用。
